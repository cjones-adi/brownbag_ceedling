# 🧪 Embedded Unit Testing - Brown Bag Learning Sessions

Welcome to the comprehensive unit testing education program for embedded system developers! This repository follows a proven **10/20/70 learning methodology** using the Ceedling framework.

## 🎯 Learning Methodology

- **10% Reading** - Foundation knowledge and concepts
- **20% Demo Sessions** - Guided demonstrations and analysis
- **70% Hands-On Experience** - Self-exploration and practice

## 📁 Directory Structure

### 📚 01-reading-materials/ (10%)
Confluence page materials covering installation, Unity, CMock, CExceptions, and the fundamental "Whats and Whys" of unit testing in embedded systems.

### 🎬 02-demo-sessions/ (20%)
Live demonstration sessions covering running tests, analyzing results, and understanding different testing patterns and scenarios.

### 🔨 03-hands-on-exercises/ (70%)
Progressive exercises designed for self-exploration, from basic first tests to complex integration scenarios.

### 📖 reference-examples/
Complete working examples:
- **simple-led-control** - Basic LED control with straightforward tests
- **complex-temp-sensor** - Comprehensive temperature sensor system with 47+ modules

### 🛠️ tools-and-templates/
Reusable templates, project scaffolding, and common testing patterns.

## 🚀 Getting Started

1. **Start with Reading Materials** - Build your foundation
2. **Attend Demo Sessions** - See testing in action
3. **Practice with Exercises** - Learn by doing (most important!)
4. **Reference Examples** - Study working implementations

## 🎯 Learning Objectives

By the end of this program, you will:

✅ Understand why unit testing prevents logic bombs in production
✅ Write tests for boundary conditions and edge cases
✅ Mock hardware interfaces to test without physical devices
✅ Test error handling and "impossible" failure scenarios
✅ Apply Test-Driven Development (TDD) to embedded projects
✅ Integrate testing into your development workflow

## 💡 Real-World Impact

> "Your compiler says your code is syntactically correct.
> Your unit tests say your code is behaviorally correct."

This program focuses on real embedded scenarios:
- Testing safety-critical logic (temperature thresholds, voltage limits)
- Developing sensor drivers before hardware arrives
- Validating error recovery paths without breaking hardware
- Testing time-dependent logic without waiting 24 hours

## 🔧 Prerequisites

- Ceedling framework installed
- Basic C programming knowledge
- Embedded development experience (beginner to expert welcome)
