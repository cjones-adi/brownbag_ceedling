# 🎬 Demo Sessions (20% of Learning)

Live demonstration sessions where you'll see unit testing in action. These sessions focus on **demonstration, running tests, and analyzing results** rather than theory.

## 🎯 Demo Session Structure

Each demo session follows this format:
1. **Problem Setup** - Real embedded scenario
2. **Live Coding** - Writing tests step-by-step
3. **Test Execution** - Running and analyzing results
4. **Discussion** - What we learned and why it matters

## 📅 Demo Session Schedule

### 🚀 Demo 1: Simple Start (30 min)
**File:** `demo-01-simple-start/`
- Setting up your first Ceedling project
- Writing your first unit test
- Understanding test output and results
- **Live Demo:** LED control module testing

### 🎭 Demo 2: Mocking Hardware (45 min)
**File:** `demo-02-mocking-hardware/`
- Introduction to CMock
- Mocking hardware interfaces (SPI, I2C, GPIO)
- Testing without physical hardware
- **Live Demo:** Sensor driver with I2C mocks

### 🔴 Demo 3: Test-Driven Development (45 min)
**File:** `demo-03-test-driven-development/`
- Red-Green-Refactor cycle
- Writing tests before implementation
- Designing testable interfaces
- **Live Demo:** Calculator module using TDD

### ⚠️ Demo 4: Boundary Conditions (45 min)
**File:** `demo-04-boundary-conditions/`
- Testing edge cases and limits
- Parametrized testing techniques
- Error injection and fault tolerance
- **Live Demo:** Temperature safety logic

### 🔗 Demo 5: Integration Testing (45 min)
**File:** `demo-05-integration-testing/`
- Testing multiple modules together
- Managing test complexity
- Integration vs unit testing trade-offs
- **Live Demo:** Complete sensor pipeline

After completing all demo sessions, you should be able to:

✅ Set up a new Ceedling project from scratch

✅ Use CMock to simulate hardware interfaces
✅ Apply TDD principles to embedded development
✅ Test boundary conditions and error scenarios
✅ Understand when to use integration vs unit tests
