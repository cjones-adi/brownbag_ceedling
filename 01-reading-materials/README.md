# 📚 Reading Materials (10% of Learning)

This section provides the foundational knowledge you need before diving into hands-on testing. These materials are designed to be consumed via Confluence pages but are outlined here for reference.

## 📖 Reading Sequence

### 1. 🛠️ Installation & Setup
- **installation-guide.md** - Complete Ceedling installation guide
- **project-setup.md** - How to configure your first testing project

### 2. 🧩 Core Framework Components
- **unity-basics.md** - Unity testing framework fundamentals
- **cmock-basics.md** - CMock for hardware interface mocking
- **ceedling-basics.md** - Ceedling build system and project structure
- **cexceptions-basics.md** - Exception handling in embedded tests

### 3. 🤔 The "Why" Behind Testing
- **why-unit-testing.md** - Real-world cases where unit tests prevent disasters
- **unit-testing-in-sdlc.md** - How unit testing fits into every SDLC phase, methodology comparison, ROI, and team adoption roadmap
- **embedded-testing-patterns.md** - Common patterns specific to embedded development

## 🎯 Learning Outcomes

After completing these readings, you should understand:

- ✅ How to install and configure Ceedling
- ✅ The role of Unity, CMock, and CExceptions
- ✅ Why unit testing is critical for embedded systems
- ✅ Common testing patterns and anti-patterns
- ✅ How testing fits into the embedded development workflow

## 📝 Key Concepts to Master

- **Test Isolation** - Each test should be independent
- **Mocking** - Simulating hardware without physical devices
- **Boundary Testing** - Testing edge cases and limits
- **Error Path Testing** - Validating failure scenarios
- **TDD Cycle** - Red-Green-Refactor development approach
